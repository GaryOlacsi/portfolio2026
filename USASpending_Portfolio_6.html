# Finding Description & impact Recommended fix
H1
Critical
Search overload: 30+ filters on first screen
No progressive disclosure. Procurement codes required before results appear. New users have no path forward.
↗ Fix
Guided mode with 6 intent tiles; advanced mode behind a toggle. Neither user pays a tax for the other.
H2
Critical
Award detail: 60 fields at equal visual weight
Award amount and compliance flags rendered identically ; no visual hierarchy, no grouping.
↗ Fix
3-zone layout: hero summary → context section → 40-field technical accordion.
H3
Critical
Filter state lost on back-navigation
Navigating to award detail and pressing back clears all filters. Users rebuild queries from scratch.
↗ Fix
Full filter state serialised in URL. Dynamic breadcrumb on every detail page.
H4
Critical
Raw dollar figures: no interpretive context
$847B means nothing without YoY delta, utilisation rate, or peer comparison. Affects all 6 archetypes.
↗ Fix
YoY delta badges, budget utilisation bars, and 5-year sparkline history on every KPI card.
H5
Major
Homepage: single omnisearch, no intent wayfinding
Four different user intents share one entry point with no routing.
↗ Fix
6 intent tiles pre-configure the destination before any searching begins.
H6
Major
Jargon used as labels without inline definitions
“IDV”, “PSC”, “obligation vs. outlay” as primary labels with no glossary or tooltip.
↗ Fix
Glossary dots on every jargon term ; hover reveals plain-English definition in place.
H7
Major
Download flow resets on back-navigation
6-step wizard with no progress indicator, no saved configurations, no file size preview.
↗ Fix
Single-page wizard: visible step track, live file size estimates, saved configurations.
H8
Major
12-column results table is uncustomisable
No column reordering, no hiding, no NAICS/PSC sort. Forced horizontal scroll.
↗ Fix
Column selection with persistence; 3 preset views for common research workflows.
H9
Minor
Chart accessibility: colour-only encoding
Canvas/SVG charts lack aria labels, keyboard navigation, and data table alternatives.
↗ Fix
aria-label on all charts; view-as-table toggle; full keyboard navigation.
H10
Minor
Mobile renders as a degraded desktop view
Technically responsive but behaviorally desktop-only. No touch-first concessions.
↗ Fix
Card-based mobile results; full-screen filter drawer; simplified sparkline charts.
💡
Key insight
H4 (no dollar context) affects all 6 user archetypes at high or medium severity; making it the single highest-impact fix and the strongest justification for the executive dashboard layer.

⚖️
Design tension: Keisha vs. Rosa
Expert needing full taxonomy exposed efficiently vs. novice needing it hidden. Progressive disclosure resolves this without compromise: guided mode for novices, advanced for experts, one toggle. Neither pays a tax for the other.
03 · Information Architecture

Five verb-based sections, one shared leaf

Replacing data-object labels with user-intent verbs. Award Detail becomes a shared leaf reachable from search, agency profiles, recipient profiles, and direct links; each preserving context via a dynamic breadcrumb.

Intent over taxonomy

Nav labels are verbs: Search, Explore, Track, Download, Learn. Users understand where to go before needing to understand the data model. Solves H5 for every non-expert user.

Progressive disclosure

Guided search first; advanced mode behind a toggle. Award detail: hero → context → technical accordion. Each mode serves its archetype without compromise.

State persistence: fixes H3

Full filter state serialised in URL; shareable, bookmarkable, back-button safe. Dynamic breadcrumb shows originating path on every detail page.

Contextual glossary: fixes H6

Glossary dots on every jargon term. Hover or tap reveals plain-English definition in place. No navigation to a separate glossary page required.

🏗
Award Detail as shared leaf: the structural answer to H3
Formally acknowledges that Award Detail is reachable from multiple paths and designs for each explicitly. The breadcrumb text changes per originating path: “← 1,204 results: HHS sole-source contracts” vs “← Dept. of Defense agency profile.” Back button always works.

07 · Key Design Decisions

Why these choices, not others

Why verb-based navigation labels?
The current site uses data object names (“Federal Account Explorer”). The redesign uses verbs: Search, Explore, Track, Download, Learn. Users understand where to go before needing to understand the data model. Solves H5 without removing anything from expert users.
Why progressive disclosure over simplification?
An early direction permanently hid advanced features. Rejected because it broke Keisha’s workflow entirely; she needs NAICS codes, PSC codes, and competition type filters without extra clicks. Progressive disclosure respects both archetypes: one toggle, no trade-offs.
Why inline glossary instead of a help center?
A separate help center requires context-switching away from the task. Inline glossary dots resolve jargon exactly where confusion occurs; on the label itself. Critical for Rosa under deadline and Patricia (the academic) who must cite her methodology in a dissertation.
Why is data completeness a feature, not a disclaimer?
The current site shows nothing when data is incomplete. A user noticing anomalously low September numbers has no explanation and loses trust. Making incompleteness visible and explained transforms a trust-eroding mystery into a trust-building transparency signal.
08 · Outcomes & Skills Demonstrated

What this project demonstrates

End-to-end UX practice on a complex, data-heavy public system. Every design decision traces back to a specific heuristic finding or persona need; documented throughout.

Systems thinking

Redesigning a complex data architecture for 6 distinct archetypes without breaking any single workflow.

Executive dashboard design

YoY delta cards, utilisation panels, sparkline histories, and contextual insight synthesis.

Progressive disclosure arch.

Guided/advanced mode-switching, 3-zone detail hierarchy, collapsible technical accordion.

Data table design

Customisable columns, preset views, inline benchmarking, and “vs prior year” columns.

State management as UX

URL-serialised filter state, dynamic breadcrumbs, and session-persistent column preferences.

Information architecture

Intent-based nav labelling, shared leaf node design, and entity cross-linking between pages.

Data visualisation

Multi-series charts, utilisation bars, sparklines; all with contextual benchmarks included.

Accessibility

Inline glossary system, aria-label strategy, and view-as-table alternative for all charts.
